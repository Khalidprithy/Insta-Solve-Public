Answers of bonus part questions

How does JavaScript work?

JavaScript is based on ECMA262 and it's just-in-time compiled language. First of all, we write JavaScript as human reading formate then the JavaScript engines (V8, Spidermonkey, chakra) the code into machine code(1 and 0). But it's a single-threaded language, so it compiles one command at a time. When we run a program it keeps all the commands in the stack heap and executes them one by one. When multiple commands are given it stores them in an event loop and uses a call back function to run them.

What does JavaScript event loop do?

We know that the JavaScript code command is executed one at a time by a single thread. So when a program is running and it passes multiple commands javascript will work on the first command first and keep all other commands in a FIFO event queue. When the first command is done compiling it will take the next command from the event queue until it's empty. But if there is an asynchronous event it will keep it aside and work on the current command. When all the task is complete then it will call the remaining asynchronous event by call back function.
